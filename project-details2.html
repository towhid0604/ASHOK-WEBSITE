Budget-Aware Neural Architecture Search for Lightweight CNNs

Overview

Neural Architecture Search (NAS) automates the design of neural network architectures, traditionally a manual and expertise-intensive process, to discover optimized models tailored to specific tasks or hardware constraints. NAS explores a vast space of possible architectures to yield networks with superior performance or efficiency compared to hand-crafted designs, especially in resource-constrained environments like mobile devices, IoT edge nodes, or wearable technology. Budget-aware methods, such as Hyperband, prioritize computational efficiency by allocating resources to promising candidates, making NAS practical for real-time applications like object detection or speech processing. This project focuses on lightweight NAS to identify compact convolutional neural networks (CNNs), comparing optimization strategies like early stopping and candidate pruning, demonstrating how automated design streamlines model development for constrained devices.

Goals

  • Define a search space for compact CNN architectures optimized for low-resource devices.
  • Implement budget-aware NAS using Hyperband to evaluate architecture candidates.
  • Compare early stopping and candidate pruning strategies for search efficiency.
  • Discover high-performing compact CNNs on benchmark datasets.
  • Analyze trade-offs in search time, computational cost, and model performance.

Approach

Define a search space with variations in CNN layers, filter sizes, and connections. Use Hyperband, which combines random search with successive halving, to evaluate candidates under computational constraints. Implement early stopping based on validation performance and prune underperforming architectures to reduce search time. Compare strategies using metrics such as search duration, computational cost, and resulting model efficiency on datasets like CIFAR-10.

Technologies Used

  • Python for scripting the search process
  • PyTorch or TensorFlow for architecture evaluation
  • Optuna or NNI (Neural Network Intelligence) for NAS implementation
  • Hyperband from scikit-optimize or custom implementation
  • Datasets from torchvision (e.g., CIFAR-10) for training and validation
